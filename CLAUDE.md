## Project Overview

A Java blockchain client library for the [Golos](https://golos.id) network (a Steem fork). It provides typed access to Golos JSON-RPC 2.0 APIs, handles transaction construction/signing, and exposes all calls as reactive streams (`Mono<T>`).

Published via JitPack. Requires Java 17.

## Architecture

### Reactive HTTP Client Pattern

All API classes extend `GolosApiReactive`, which holds a pre-configured Spring WebFlux `WebClient`. Every public method wraps a JSON-RPC 2.0 call:

```java
// Pattern used across all API classes:
ApiMethod method = new ApiMethod("api_name", "method_name", params);
return webClient.post().bodyValue(method).retrieve()
    .bodyToMono(SomeApiResponse.class);
```

`GolosApiConfiguration` wires the `WebClient` with the URL from `application.properties` and Jackson `SNAKE_CASE` naming strategy.

### Transaction Building

Transactions are built with `TransactionBuilder` (prototype-scoped Spring bean). Always obtain it via `TransactionFactory` (never inject `TransactionBuilder` directly):

```java
// Correct usage:
TransactionBuilder builder = transactionFactory.getBuilder();
builder.addOperation(op).buildAndSign(wif);
```

`SecurityUtils` handles ECDSA signing using BouncyCastle + the `eosiojava` library. The README notes this is slow because it delegates to a JS library under the hood.

### Model Structure

- `model/` — top-level DTOs (`Account`, `Block`, `Asset`, `Transaction`, `Authority`, `Content`, `Price`)
- `model/api/` — `ApiMethod` (JSON-RPC request) and typed `ApiResponse` wrappers
- `model/op/` — concrete blockchain operation classes (Transfer, Vote, Comment, Custom, etc.)
- `model/op/virtual/` — virtual operations generated by the blockchain (rewards, fills, interest)
- `model/nft/` — NFT token and collection models
- `model/exchange/` — limit order and exchange query/result models
- `types/` — primitive blockchain types
- `utilities/` — helper utilities

```properties
golos.api=https://apibeta.golos.today

spring.jackson.property-naming-strategy=SNAKE_CASE
```

### Serialization

Jackson is configured globally with `SNAKE_CASE` naming so Java `camelCase` fields map automatically to JSON `snake_case`. Some complex types (operation polymorphism, asset precision) have custom deserializers.

## Test Notes

Integration tests in `GolosApiTest` hit the live `apibeta.golos.today` endpoint and use hardcoded test WIF keys. Many test methods are commented out. `SecurityUtilsTest` and `MarketHistoryApiTest` are the main focused unit/integration tests.
